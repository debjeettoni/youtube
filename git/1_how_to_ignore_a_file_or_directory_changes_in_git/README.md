### **What is `.gitignore` file?**

A `.gitignore` file is a plain text file where you list files or directories that you want Git to ignore. Ignored files are not tracked by Git, meaning they won't show up in `git status` and won't be committed to your repository. This is perfect for log files, build artifacts, and secret credentials.

```bash
# This is a comment. Lines starting with # are ignored.

# Ignore a specific file by name
secret.key
config.ini

# Ignore all files with the .log extension
*.log

# Ignore a specific directory and everything inside it
/node_modules/
/build/

# You can also ignore files in any subdirectory
# This will ignore 'debug.log' in the root, in 'src/', etc.
debug.log

# Use ! to negate a pattern.
# Ignore all .log files, EXCEPT for important.log
*.log
!important.log
```

---

### **Example 1 - Ignore a specific single file**

To ignore a single file, simply add its name to the `.gitignore` file.

This is useful for configuration files that contain sensitive information or local settings that shouldn't be shared with the team.

```bash
# .gitignore

# Ignore the local database configuration file
db_config.local
```

---

### **Example 2 - Ignore a specific single directory**

To ignore an entire directory and all of its contents, add the directory name followed by a `/`.

This is commonly used for folders containing dependencies, build outputs, or user-uploaded content.

```bash
# .gitignore

# Ignore the directory containing all node dependencies
node_modules/

# Ignore the build output directory
dist/
```

---

### **Example 3 - Ignore multiple files using regular expression**

You can use wildcard patterns to ignore multiple files that share a common naming convention. The asterisk `*` matches zero or more characters.

This is highly effective for cleaning up temporary files, logs, or compiled code.

```bash
# .gitignore

# Ignore all files ending with .log
*.log

# Ignore all temporary files ending with .tmp
*.tmp

# Ignore files generated by macOS
.DS_Store
```

---

### **Example 4 - Ignore multiple directories using regular expression**

Wildcards can also be used to ignore multiple directories.

This is useful if your project generates several log or build directories with varying names.

```bash
# .gitignore

# Ignore any directory ending with "-cache"
*-cache/

# Ignore any directory inside the 'logs/' folder
logs/*/
```

---

### **Example 5 - Ignore a directory but keep a single file (Negation)**

Sometimes you need to ignore a whole directory but make an exception for one or more files inside it. You can achieve this using the `!` prefix.

**Important:** You cannot re-include a file if its parent directory is excluded. First, exclude everything, then re-include the specific file.

```bash
# .gitignore

# 1. Ignore the entire 'logs' directory
logs/

# 2. BUT do NOT ignore the important.log file inside it
!logs/important.log
```

---

### **`.gitignore` hierarchy and precedence**

Git applies ignore rules from multiple sources. Understanding the order of precedence is key:

  * **Command Line:** Rules passed directly to Git commands have the highest precedence.
  * **.git/info/exclude:** This file contains rules for your local repository only. They are not shared with the team.
  * **.gitignore (local):** Files in your repository. The rules in a subdirectory's `.gitignore` file have precedence over those in a parent directory. This is the most common way to share ignore rules.
  * **Global .gitignore:** A file you configure on your system to apply to *all* of your Git repositories (`git config --global core.excludesfile ~/.gitignore_global`).

Rules are processed from bottom to top, with later rules overriding earlier ones for the same path.

---

### **How to ignore files already tracked by Git (`git rm --cached`)**

What if you accidentally committed a file and now want to ignore it? Adding it to `.gitignore` isn't enough because Git is already tracking it.

You need to tell Git to stop tracking the file, but keep it in your local directory.

**Step 1:** Add the file path to your `.gitignore` file.
**Step 2:** Run the following command:

```bash
# Tell Git to remove the file from the staging area (the index)
# but leave the actual file in your working directory.
# --cached is the key!

git rm --cached secret_api_key.txt
```

**Step 3:** Commit this change. The file will now be ignored for all future commits.
